# Column
 The Column Class provides properties, methods, and events to interface with the HTML tags generated by the [silk:Column](../tags/Column.md) tag.
This Object is used internally by the [silk:Table](../tags/Table.md) component and the [Table](Table.js.md) Class to create a table's columns. It can also be used directly to add columns to a table using JavaScript dynamically.

**Kind**: global class  


## Constructor
 <a name="_new"></a>

### new Column(id, option)
Returns a Column instance.


| Param | Type | Description |
| --- | --- | --- |
| id | <code>String</code> | Column's unique identifier |
| option | <code>Object</code> | Object configuration. |
| [option.title] | <code>Object</code> | The text shown in the column header |
| [option.tableID] | <code>Object</code> | The ID of the target table |
| [option.template] | <code>Object</code> | HTML template containing the data binding fields. |
| [option.width] | <code>Object</code> | The column's width |
| [option.align] | <code>Object</code> | Horizontal alignment: left, center, and right. |
| [option.valign] | <code>Object</code> | Vertical alignment: top, middle, and bottom |
| [option.wrap] | <code>Object</code> | Defines if the text will wrap. True by default. |
| [option.rendererFn] | <code>Object</code> | The name of the function that will render the column's content. |
| [option.targetPage] | <code>Object</code> | The name of the page, or modal, which will be shown when the column is clicked. |
| [option.cssClass] | <code>Object</code> | The CSS class or classes to style the column's content. |
| [option.draggable] | <code>Object</code> | Defines if the column's rows can be dragged. False by default. |
| [option.ordering] | <code>Object</code> | When set to "true," the column header allows data ordering or sorting. |
| [option.inputType] | <code>Object</code> | The input type rendered when the table is in input mode. |
| [option.inputMask] | <code>Object</code> | The mask defining the input mode. |
| [option.valueColumn] | <code>Object</code> | The name of the data column that will fill the input. |
| [option.labelColumn] | <code>Object</code> | The name of the data column used to define the input's label. |
| [option.visibleColumn] | <code>Object</code> | This property determines the input's visibility. |
| [option.sortColumn] | <code>Object</code> | The column used to sort the dataProvider. |
| [option.idColumn] | <code>Object</code> | Defines the input's identifier to be used to extract the entered data. |
| [option.cleanIf] | <code>Object</code> | Logic statement indicating the condition in which the column cell should show empty. |
| [option.distinct] | <code>Object</code> | The column that will determine the distinct condition. |

## Events
### on:visible
This event is triggered when the column is rendered. If the event's function returns true, the column will be added. If it returns false, it will not be added.
Created with the ```Column.on("visible", function(item){})``` method.

**Kind**: event emitted by [<code>Column</code>](#Column)
